Pipeline
05

How It Works

01
Sensor Data Ingestion
Ultrasonic IoT sensors inside each bin transmit fill-level data every 5 minutes via MQTT to the central server. Fill >60% triggers active-node status.
02
Graph Construction
City streets are modeled as weighted directed graphs using OpenStreetMap data. Edge weights reflect distance, traffic density, and road class.
03
Route Computation
Dijkstra's algorithm resolves shortest paths between all active nodes. A TSP nearest-neighbor heuristic then sequences stops for minimum total distance.
04
Live Dispatch + GPS
Optimized routes are pushed to driver apps with real-time GPS. If a new critical bin emerges mid-route, the system recalculates and pushes an update instantly.
